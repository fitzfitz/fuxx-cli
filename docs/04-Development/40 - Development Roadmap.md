# 40 - Development Roadmap

The ordered path from nothing to a shipped v1. Each milestone is its own note with a
status field and a checklist. Update status as you go; the [[60 - Progress Log]] records
what actually happened each session.

## The rule of one milestone at a time

Finish and verify each milestone before starting the next. "Verify" means the milestone's
own success check passes. Record it in [[60 - Progress Log]].
